# SaaS Onboarding High-KPI

Expert skill for designing and building SaaS onboarding flows that maximize activation, reduce time-to-first-value, and boost trial-to-paid conversion.

## When to Use

- Redesigning or auditing an existing onboarding flow
- Building onboarding from scratch for a new SaaS product
- Improving activation rate, TTFV, or trial-to-paid conversion
- Generating onboarding UI components and analytics instrumentation

## 2026 Benchmarks

| Metric                          | Average SaaS | Top Quartile | Recommended Target |
|---------------------------------|--------------|--------------|--------------------|
| Time-to-First-Value (TTFV)     | 7-36 hours   | < 5-15 min   | < 10 minutes       |
| Activation Rate                | 25-40%       | 55-70%+      | > 60%              |
| Onboarding Completion Rate     | 50-70%       | 85%+         | > 85%              |
| Trial-to-Paid Conversion       | 8-25%        | 30-50%+      | > 35%              |
| 7-Day Retention                | 40-50%       | 65%+         | > 65%              |

## Implementation Roadmap

1. **Measure baseline** — Instrument current TTFV and drop-off rates before changing anything.
2. **Define the Aha Moment** — Identify the single action that correlates with long-term retention.
3. **Reduce to 5-7 steps** — Cut every step that doesn't directly lead to the Aha Moment.
4. **Add checklist + quick wins** — Interactive, actionable items with the first one pre-checked.
5. **Personalize lightly** — Segment by role, team size, or traffic source (2-3 paths max).
6. **Instrument and A/B test** — Track every step, run experiments on copy, order, and friction.
7. **Iterate weekly** — Review funnel data, fix the biggest drop-off, repeat.

## Usage Examples

| Prompt | What the Skill Does |
|--------|---------------------|
| "Redesign our onboarding to increase activation from 35% to 60%" | Diagnoses current flow, proposes a new 3-5 step flow, generates component code, and sets up analytics events |
| "Create a 3-step value-first onboarding flow" | Generates a stepper component with copy, checklist, and instrumentation |
| "Audit this onboarding flow and suggest improvements" | Runs the quick audit checklist, identifies gaps, proposes fixes with expected KPI impact |
| "Generate a personalized onboarding checklist for different user segments" | Produces a checklist component with segment-aware items and tracking |
